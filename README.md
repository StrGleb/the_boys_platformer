# The Boys

2D platformer на `C++` и `SFML`, сделанный как учебный проект. Игрок управляет Бутчером, проходит 3 уровня, прыгает по платформам, избегает шипов, побеждает врагов и в финале должен добраться до машины после победы над Homelander.

## Что есть в игре

- главное меню с кнопками `PLAY` и `EXIT`
- 3 уровня, которые загружаются из текстовых файлов
- обычные враги `Vought Agent`
- финальный враг `Homelander` на 3 уровне
- экран прохождения уровня
- экран финальной победы
- камера, следующая за игроком
- параллакс-фон города

## Управление

Сейчас управление сделано на мыши:

- `ЛКМ` - движение влево
- `ПКМ` - движение вправо
- `Средняя кнопка мыши / колесо` - прыжок

### Почему управление на мыши

Проект редактируется в `VS Code` на Windows, а собирается и запускается через `WSL`. В такой связке нажатия клавиш в окне SFML иногда передавались некорректно, из-за чего персонаж не реагировал стабильно. Поэтому для демонстрационной версии управление временно переведено на мышь, так как события кнопок мыши в этой конфигурации работают надёжнее.

Если запускать проект в обычной Linux-среде или полностью на одной системе, управление можно без проблем вернуть на клавиатуру.

## Цель игры

- На `1` и `2` уровне нужно добраться до машины в конце карты.
- На `3` уровне нужно сначала победить `Homelander`, и только после этого машина засчитает прохождение.

## Структура проекта

```text
src/
  main.cpp
  core/
    Game.h
    Game.cpp
  entities/
    Player.h
    Player.cpp
    Enemy.h
    Enemy.cpp
  world/
    Map.h
    Map.cpp
  ui/
    Menu.h
    Menu.cpp

levels/
  level1.txt
  level2.txt
  level3.txt
  level_guide.txt

assets/
  *.png
```

### Кто за что отвечает

- `Game` - главный класс, управляет состояниями игры, уровнями, врагами, событиями и отрисовкой
- `Player` - движение игрока, прыжок, анимации, столкновения с картой
- `Enemy` - логика обычных врагов и Homelander
- `Map` - загрузка уровня из `.txt` и отрисовка блоков
- `Menu` - главное меню и визуал кнопок

## Формат уровней

Каждый уровень хранится в обычном текстовом файле. Один символ = один тип клетки.

- `0` - пустота
- `1` - бетон
- `2` - кирпичная стена
- `3` - финиш, машина
- `4` - шипы
- `e` / `E` - агент Vought
- `h` / `H` - Homelander

Это позволяет быстро редактировать уровни без изменения кода.

## Сборка

### Вариант 1: через VS Code

В проекте уже настроен `tasks.json`, поэтому можно использовать стандартную сборку `Run Build Task`.

### Вариант 2: вручную

```bash
g++ -std=c++20 -g -I./src \
  ./src/main.cpp \
  ./src/core/Game.cpp \
  ./src/entities/Player.cpp \
  ./src/entities/Enemy.cpp \
  ./src/world/Map.cpp \
  ./src/ui/Menu.cpp \
  -o ./game.out \
  -lsfml-graphics -lsfml-window -lsfml-system
```

Запуск:

```bash
./game.out
```

## Зависимости

- `g++` с поддержкой `C++20`
- `SFML`

## Особенности реализации

- уровни отделены от кода и лежат в `levels/`
- враги создаются по маркерам из файла уровня
- фон и интерфейс рисуются отдельно от игрового мира
- для внутренних констант и вспомогательных функций в `.cpp` используется анонимный `namespace`
